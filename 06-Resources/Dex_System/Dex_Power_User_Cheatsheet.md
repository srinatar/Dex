# Dex Power User Cheatsheet

**Quick reference for commands and when to use them.**

---

## Daily Workflow

| Command | When to use |
|---------|--------------|
| `/daily-plan` | Start of day — calendar + tasks + priorities + meeting prep in one shot |
| `/review` or `/daily-review` | End of day — capture learnings, track completion, surface follow-ups |

---

## Weekly Workflow

| Command | When to use |
|---------|--------------|
| `/week-plan` | Start of week — set Top 3 priorities, plan around calendar shape |
| `/week-review` | End of week — what shipped, patterns, goal progress |

---

## Quarterly (Optional)

| Command | When to use |
|---------|--------------|
| `/quarter-plan` | Start of quarter — set 3–5 strategic goals |
| `/quarter-review` | End of quarter — completion, learnings, evidence capture |

---

## Meetings

| Command | When to use |
|---------|--------------|
| `/meeting-prep [person]` | Before a meeting — full context: past meetings, open items, relationship notes |
| `/process-meetings` | After meetings — extract summaries, action items, update person pages |
| `/triage` | Inbox overflow — route orphaned files, extract scattered tasks |

---

## Projects

| Command | When to use |
|---------|--------------|
| `/project-health` | Anytime — see what’s on track, stalled, or blocked across all projects |
| `/product-brief` | New product idea — guided questions → PRD draft |

---

## Tasks (Natural Language)

| Say this | What happens |
|----------|--------------|
| "Create a task to review Q1 numbers" | Task created, pillar inferred, syncs to Tasks.md |
| "I finished X" / "Mark Y as done" | Task completion found and marked done everywhere |
| "What’s my P0 tasks?" | Work MCP surfaces priority-filtered tasks |

---

## Career Development

| Command | When to use |
|---------|--------------|
| `/career-setup` | One-time — initialize career system |
| `/career-coach` | Weekly check-ins, self-reviews, promotion readiness |
| `/resume-builder` | Build resume / LinkedIn via guided interview |

---

## System & Discovery

| Command | When to use |
|---------|--------------|
| `/dex-level-up` | Find unused features based on your usage |
| `/dex-whats-new` | Check system updates + Claude changelog |
| `/dex-backlog` | AI-ranked improvement ideas |
| `/dex-improve` | Turn an idea into an implementation plan |
| `/dex-update` | Update Dex (safe, shows diff first) |
| `/dex-rollback` | Undo last update if something broke |
| `/xray` | See what just happened — context, MCPs, hooks |

---

## Integrations

| Command | When to use |
|---------|--------------|
| `/integrate-mcp` | Connect tools (Jira, Confluence, GitHub, etc.) from Smithery |
| `/getting-started` | Post-onboarding tour, adaptive to your setup |
| `/ai-setup` | Configure budget cloud models or offline mode |

---

## Quick Paths by Role

**Product Manager**
- Morning: `/daily-plan`
- Before stakeholder call: `/meeting-prep [name]`
- Weekly: `/project-health`, `/week-plan`, `/week-review`
- New idea: `/product-brief`

**Sales / AE**
- Before call: `/meeting-prep [contact]`, `/call-prep`
- Pipeline: `/deal-review`, `/pipeline-health`
